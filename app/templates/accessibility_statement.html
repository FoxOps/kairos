Déclaration d'accessibilité

Engagement de Leviia en faveur de l'accessibilité numérique

Notre engagement

Chez Leviia, nous nous engageons à rendre notre application accessible à tous, y compris aux personnes en situation de handicap. Nous travaillons activement pour améliorer l'accessibilité de notre plateforme conformément aux normes WCAG 2.1 niveau AA.

Conformité aux normes

Cette application vise à être conforme aux Web Content Accessibility Guidelines (WCAG) 2.1 au niveau AA. Ces directives définissent les exigences pour rendre le contenu web plus accessible aux personnes handicapées.

Nous avons implémenté les critères suivants pour garantir une expérience accessible :

  • Structure sémantique : Utilisation de balises HTML appropriées (h1, h2, landmarks, etc.).
  • Navigation au clavier : Tous les éléments interactifs sont accessibles via le clavier.
  • Contraste des couleurs : Respect des ratios de contraste minimum (4.5:1 pour le texte).
  • Attributs ARIA : Utilisation d'attributs ARIA pour améliorer l'accessibilité des composants dynamiques.
  • Messages d'erreur accessibles : Les erreurs sont clairement annoncées aux lecteurs d'écran.
  • Support des préférences utilisateur : Respect des préférences comme prefers-reduced-motion et prefers-color-scheme.

Technologies utilisées

Cette application utilise les technologies suivantes :

  • HTML5 : Structure sémantique et balises accessibles.
  • CSS3 : Styles respectant les bonnes pratiques d'accessibilité.
  • JavaScript : Fonctionnalités interactives accessibles via le clavier.
  • Flask : Framework backend pour la génération des pages.
  • FullCalendar : Composant de calendrier accessible.

Limitations et alternatives

Malgré nos efforts, certaines parties de l'application peuvent ne pas être entièrement accessibles. Voici les limitations connues et les alternatives proposées :

  • Composants tiers : Certains composants externes (comme FullCalendar) peuvent avoir des limitations. Nous travaillons avec leurs équipes pour améliorer leur accessibilité.
  • Contenu généré par les utilisateurs : Le contenu ajouté par les utilisateurs (ex : descriptions de shifts) peut ne pas être entièrement accessible. Nous encourageons les utilisateurs à suivre les bonnes pratiques.

Si vous rencontrez un problème d'accessibilité, veuillez nous en informer via les coordonnées ci-dessous.

Feedback et contact

Nous accueillons favorablement vos retours sur l'accessibilité de notre application. Si vous rencontrez des difficultés ou avez des suggestions, veuillez nous contacter :

  • Email : accessibility@leviia.example.com
  • Formulaire de contact : Utilisez le formulaire de contact disponible dans l'application.

Nous nous engageons à répondre à vos préoccupations dans les 5 jours ouvrables.

Amélioration continue

L'accessibilité est un processus continu. Nous nous engageons à :

  • Effectuer des audits réguliers de l'accessibilité.
  • Corriger les problèmes identifiés dans les meilleurs délais.
  • Former notre équipe aux bonnes pratiques d'accessibilité.
  • Intégrer des tests d'accessibilité dans notre processus de développement.
