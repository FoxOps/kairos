Comment ça marche ?

  1. Définissez l'ordre de rotation : Attribuez une position (1, 2, 3, ...) à chaque utilisateur pour définir l'ordre dans lequel ils seront désignés pour les astreintes.
  2. Cochez les utilisateurs : Sélectionnez quels utilisateurs inclure dans la rotation.
  3. Choisissez la période : Sélectionnez la date de début et de fin pour la génération.
  4. Prévisualisez : Utilisez le bouton "Prévisualiser" pour voir ce qui sera généré sans sauvegarder.
  5. Générez : Cliquez sur "Générer les astreintes et shifts" pour créer les astreintes et les shifts en base de données.

Règles appliquées :

  • Chaque astreinte commence selon la configuration (par défaut vendredi à 21h) et se termine selon la configuration (par défaut vendredi suivant à 07h).
  • La rotation suit l'ordre que vous avez défini et sauvegardé. Cet ordre est stocké dans la configuration et sera réutilisé pour les prochaines générations.
  • Si un utilisateur n'est pas disponible (congé, astreinte existante), le système passe au suivant dans la rotation.
  • Un message d'avertissement est affiché si aucun utilisateur n'est disponible pour une période.
  • Les shifts sont générés automatiquement après les astreintes selon les nouvelles règles métiers (13h-21h pour l'astreinte, rotation des créneaux, etc.).

Note : L'ordre de rotation est sauvegardé automatiquement dans la base de données lorsque vous cliquez sur "Sauvegarder l'ordre".
