## 🎉 **WHAT WAS FIXED THIS SESSION**

Every issue listed in "Remaining work" from the previous session was addressed:
132 failing tests brought down to 0. Breakdown by commit (see `git log` on the branch):

1. **`config.py` / `app/config/base.py`**: added the missing
   `LOGIN_DISABLED`, `REMEMBER_COOKIE_DURATION`, `SESSION_PROTECTION` keys
   (standard Flask-Login keys). The test comparing `SECRET_KEY` to a static
   value was fixed to reflect the actual behavior (random secret
   generated by default, safer than a hardcoded value).

2. **`app/routes/main.py`**: several `url_for("leave")`, `url_for("add_shift")`,
   etc. had lost their blueprint prefix (`main.`) during a
   previous file reorganization → cascading `BuildError`, including
   in templates (`{% set route = 'leave' %}` in `_pagination.html`,
   used by `leave.html`/`oncall.html`/`schedule.html`).

3. **`can_add_shift` / `can_add_oncall` / `can_add_leave`**
   (`app/utils/helpers/common_helpers.py`): the functions didn't yet handle
   basic business rules (working days, Friday 9pm for on-call
   rotations, overlapping leave, invalid dates). The calling routes
   were additionally passing a `user_id` (int) instead of a `User` object (`user.is_authenticated`
   crashed). Both ends were fixed and aligned on a simple `bool`
   return (no `(bool, message)` tuple).

4. **`OnCallAutomation` / `ShiftAutomation`**
   (`app/utils/automation/oncall_automation.py`, `shift_automation_class.py`):
   implemented the methods that didn't yet exist and that the tests
   already expected: `check_oncall_constraint`, `find_next_available_user`,
   `can_assign_shift`, `find_replacement_user`, and rewrote
   `generate_oncall_schedule`/`generate_shift_schedule` to return real
   `OnCall`/`Shift` objects (genuine dry_run) instead of dicts.

5. **Error handling and logging** (`app/__init__.py`,
   `app/utils/logging/logger.py`): the `app/utils/logging` module only
   provided basic logging config. Added `SensitiveDataFilter` (masks
   password/token/api_key in logs), dedicated `http_errors` and
   `audit` loggers with files under `logs/`, `log_http_error`,
   `get_error_template_data`, `log_audit_action`, and registered real
   Flask error handlers (400/401/403/404/405/500/502/503/504 + ValueError/TypeError)
   instead of letting Flask fall back to its default pages.

6. **Recurring test bugs** found in several files (`test_decorators.py`,
   `test_helpers.py`, `test_routes.py`, `test_ics_export.py`): `app`
   fixture used without being imported/declared (`NameError`), incorrect
   `second_user` login password (`second123` instead of `test123`,
   didn't match the hash created by the fixture), dead assertions
   referencing a `message` variable that was never defined (leftover from an
   earlier bool/tuple refactor), "requires_admin" tests using `logged_in_client`,
   which is... an admin (added a dedicated `non_admin_client` fixture).

7. **`tests/conftest.py`**: the `test_shift` fixture hardcoded
   `shift_type_id=1` without ever creating a `ShiftType` with that id (SQLite
   doesn't enforce the FK constraint by default) → 404 on any route that
   reloads the `ShiftType` by that id. Now depends on `test_shift_type`.

8. **Frontend**: duplicated CSS utility classes `.gap-*`/`.min-w-*` in
   `dark-theme.css` (only existed in `base-styles.css`), missing
   accent in the `aria-label` of the theme toggle button.

9. **HTTPS redirect loop in prod**: `create_app()` always loads
   `app.config.Config` (never `ProductionConfig`) regardless of `FLASK_ENV`,
   yet only `ProductionConfig` read `TALISMAN_FORCE_HTTPS` from the
   environment → the variable was ignored in real deployment, Talisman forced
   an `https://` redirect even though no reverse proxy was terminating TLS.
   Moved into `Config` (the class actually loaded), default `False`. Added
   `ProxyFix` for correct scheme detection behind a real
   TLS reverse proxy.

10. **Docker**: `entrypoint.sh` attempted `chown` while already running as a
    non-root UID (`user: "1000:1000"` in `docker-compose.yml`) → consistent
    failure, container dead before flask/gunicorn even started.
    `DATABASE_URL=sqlite:///app.db` (relative) resolved into
    `app.instance_path` instead of `/app/data` (the mounted volume) → the DB
    was never persisted to the host.

11. **OIDC**: `app/routes/auth.py` called nonexistent methods on
    `OIDCAuthLib` (generic Authlib API instead of the internal API actually
    implemented) → every OIDC login crashed with `AttributeError`. Added
    `OIDC_INTERNAL_ISSUER` for the case where the OIDC provider is
    only reachable by the container internally (e.g. same docker-compose) but
    must remain reachable by the browser via a different address.

## ✅ **Services/repositories split completed** (session continuation)

`app/services/` and `app/repositories/` were empty scaffolding
(`import app.services`/`app.repositories` raised `ImportError`) and
`app/routes/main.py` (1287 lines) had never been split as planned
in the initial Phase 2 plan. Done:

- **`app/repositories/`**: `UserRepository`, `GroupRepository`,
  `ShiftRepository`, `ShiftTypeRepository`, `OnCallRepository`,
  `LeaveRepository` - pure data access (SQLAlchemy queries)
- **`app/services/`**: `UserService`, `ShiftService`, `OnCallService`,
  `LeaveService`, `ExportService`, + `ScheduleService` (new, calendar
  aggregation) - business logic (`can_add_*` validations, automatic
  rebalancing after leave), built on top of the repositories
- **`app/routes/main.py`** split into `dashboard_routes.py`,
  `shift_routes.py`, `oncall_routes.py`, `leave_routes.py`: each file
  imports `main_bp` from `main.py` and registers its routes onto it - the
  blueprint stays named `"main"` everywhere, so none of the ~300
  `url_for("main.xxx")` references (templates + tests) had to change. Routes now
  only parse the request, call the service, and turn the
  result into flash/redirect/JSON

Verified: full suite still at 504/504, Flask route map
identical before/after, Docker container rebuilt and tested under real
conditions (login, all pages, JSON API, full
add/delete cycle for shift + on-call + leave with automatic rebalancing
triggered).
